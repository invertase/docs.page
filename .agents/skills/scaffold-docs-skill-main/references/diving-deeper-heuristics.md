# Diving Deeper heuristics

Use these tests to decide whether a topic belongs in Getting Started (GS), Diving Deeper (DD), or Reference (Ref).

## A topic belongs in Diving Deeper if any of these hold

- **Aside test**: while drafting Getting Started you'd be tempted to digress on it but cut for flow.
- **Alternative-path test**: another way to do something Getting Started covers (sync vs. async, alternate backend, alternate entry point).
- **Customization test**: something a reader reaches for after the tutorial to tune behavior: configuration surfaces, hooks, lifecycle overrides, options.
- **Auxiliary-use-case test**: a use case that didn't win Phase 1 but is common enough to warrant a write-up.
- **Mental-model test**: a subsystem with internal structure where the *why* matters for correct use.

## A topic belongs in Reference only if

A signature plus a one- or two-sentence purpose covers it.

## The operational test

If you find yourself wanting more than a sentence or two of *intent* for the topic, promote it to Diving Deeper. This applies both during initial Phase 1 tagging and during Phase 4 module drafting (see "kickback rule" in `phases/4-reference.md`).
