Write documentation as Markdown or MDX under `docs/`. Each file becomes a page; `docs.json` controls how pages appear in the sidebar.

## Pick a page layout

Before you draft, decide what the reader needs from the page — a first walkthrough, steps for one task, lookup facts, or background on *why*. These **page templates** follow the [Diátaxis](https://diataxis.fr/) framework. They are layouts for individual `.mdx` files, not site themes or `docs init` scaffolds.
