docs.page builds site navigation from `docs.json`. Sidebar order also drives automatic previous/next links unless frontmatter overrides them.

## Sidebar structure

The `sidebar` field is an array of group objects. Each group can contain pages, nested groups, icons, and tab scoping.

```json
{
  "sidebar": [
    {
      "group": "Getting Started",
      "tab": "root",
      "pages": [
        { "title": "Overview", "href": "/", "icon": "book" },
        { "title": "Setup", "href": "/setup", "icon": "rocket" }
      ]
    },
    {
      "group": "Advanced",
      "pages": [
        {
          "group": "Deployment",
          "pages": [
            { "title": "Custom Domains", "href": "/custom-domains" }
          ]
        }
      ]
    }
  ]
}
```

### Page item fields

| Field | Required | Description |
| --- | --- | --- |
| `title` | yes | Label in the sidebar |
| `href` | yes | Root-relative path (maps to `docs/{path}.mdx`) |
| `icon` | no | Icon name for the sidebar entry |

### Group fields

| Field | Description |
| --- | --- |
| `group` | Section label |
| `tab` | Restrict visibility to a tab `id` from the `tabs` array |
| `href` | Optional link for the group header |
| `icon` | Optional group icon |
| `pages` | Nested pages or sub-groups |

Keep nesting to three levels: group ﻗ│φ sub-group ﻗ│φ page.

## Tabs {#tabs}

Top-level tabs split documentation into sections (for example, Documentation, Components, CLI).

```json
{
  "tabs": [
    { "id": "root", "title": "Documentation", "href": "/" },
    { "id": "cli", "title": "CLI", "href": "/cli" }
  ]
}
```

Assign `tab` on sidebar groups to scope them to a specific tab. Tabs support locale-aware routing when using multi-locale sidebars.

## Multi-locale documentation

Use a locale-keyed object instead of a flat array for `sidebar`:

```json
{
  "sidebar": {
    "default": [
      { "group": "Guides", "pages": [{ "title": "Start", "href": "/" }] }
    ],
    "fr": [
      { "group": "Guides", "pages": [{ "title": "Dﺃﺏmarrer", "href": "/" }] }
    ]
  }
}
```

Locale keys (except `default`) populate the `locales` array at parse time. The locale switcher UI lets readers switch between locale-specific sidebars. Anchors, tabs, and header links can also scope to a locale.

## Previous and next links

Previous/next navigation appears at the bottom of each page.

**Automatic inference** (default): links follow sidebar order when `content.automaticallyInferNextPrevious` is `true`.

**Frontmatter override**: set `previous` and `next` in page frontmatter to explicit paths.

```mdx
---
title: My Page
previous: /getting-started
next: /configuration
---
```

## Header anchor links

The `anchors` array adds icon links to the documentation header:

```json
{
  "anchors": [
    { "title": "GitHub", "href": "https://github.com/org/repo", "icon": "github" }
  ]
}
```

Anchors support optional `locale` and `tab` scoping.
