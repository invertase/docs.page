docs.page reads `docs.json` and `docs/**/*.mdx` from a public GitHub repository. This tutorial walks you through scaffolding a project and previewing it locally.

## Prerequisites

- A public GitHub repository (private repos are blocked)
- Node.js 18+ for the CLI and local preview

## Steps

<Steps>
  <Step title="Scaffold the project">
    Run the CLI at your project root to create `docs.json` and starter pages:

    ```bash
    npx @docs.page/cli init
    ```

    The command creates:

    - `docs.json` — site configuration
    - `docs/index.mdx` — landing page
    - `docs/usage.mdx` — sample content page
  </Step>

  <Step title="Customize docs.json">
    Open `docs.json` and set your project name and description:

    ```json
    {
      "name": "My Project",
      "description": "Documentation for My Project"
    }
    ```

    Add sidebar entries as you create more pages. See [Navigation](/navigation) for sidebar structure.
  </Step>

  <Step title="Write your first page">
    Edit `docs/index.mdx` with frontmatter and content:

    ```mdx
    ---
    title: Welcome
    description: Getting started with My Project
    ---

    # Welcome

    Your documentation lives here as MDX files.
    ```
  </Step>

  <Step title="Preview locally">
    Open [https://docs.page/preview](https://docs.page/preview) and select your project directory. The preview connects to a local WebSocket dev server for live MDX editing and hot reload — no GitHub push required.

    See [Local Preview](/local-preview) for details.
  </Step>

  <Step title="Publish to GitHub">
    Commit `docs.json` and `docs/` to a **public** repository, then visit:

    ```
    https://docs.page/{owner}/{repo}
    ```

    Replace `{owner}` and `{repo}` with your GitHub org or user and repository name.
  </Step>
</Steps>

## Verify

Open your published URL. You should see your landing page with the project name from `docs.json` in the header.
