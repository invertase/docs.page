Documentation pages live in `docs/` as `.mdx` files. docs.page compiles MDX into a document intermediate representation (IR) with syntax-highlighted code blocks, sanitized HTML, and registered component nodes.

## Page frontmatter

Every page should include at minimum:

```mdx
---
title: Page title
description: One sentence for search and social cards
---
```

Optional frontmatter fields include `image`, `previous`, `next`, and `redirect`.

## MDX components

docs.page renders built-in shortcode components:

| Component | Purpose |
| --- | --- |
| `Accordion` | Collapsible sections for long reference content |
| `Callout` | Info, warning, and error callouts |
| `Card` / `Cards` | Linked content cards |
| `CodeGroup` | Tabbed code block groups |
| `Image` | Optimized documentation images |
| `Icon` | Inline icons |
| `Property` | Config field reference blocks |
| `Steps` / `Step` | Sequential tutorial steps |
| `Tabs` / `TabItem` | Tabbed content panels |
| `Tweet` | Embedded tweets |
| `Video`, `YouTube`, `Vimeo` | Embedded video players |

See the [Components](/components) tab for usage examples.

## Variables {#variables}

Inject values from `docs.json` into MDX with `{{ dotted.path }}` placeholders:

```json
{
  "variables": {
    "api": {
      "baseUrl": "https://api.example.com"
    }
  }
}
```

```mdx
Call `{{ api.baseUrl }}/v1/users` to list users.
```

Substitution runs at bundle time using the `variables` object from your configuration.

## GitHub-hosted assets

Relative image and asset paths resolve to `raw.githubusercontent.com` URLs for the active ref or base branch. Store assets in your repository and reference them with relative paths:

```mdx
![Architecture diagram](./images/architecture.png)
```

## Page redirects

Set `redirect` in frontmatter to send readers elsewhere:

```mdx
---
title: Old Page
redirect: /new-page
---
```

Redirects support internal paths and external URLs. Ref-aware base resolution applies when redirecting within the same documentation site.

## Raw markdown access

Append `.md` or `.mdx` to any published page URL to retrieve unrendered source:

```
https://docs.page/{owner}/{repo}/writing-content.md
```

## Content display options

Control page chrome via `content` in `docs.json`:

- `headerDepth` — table of contents depth (default `3`)
- `zoomImages` — click-to-zoom on images
- `showPageTitle` / `showPageImage` — toggle title and hero image

See [docs.json Reference](/configuration#content-display).
