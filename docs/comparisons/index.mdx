You are picking a home for documentation — not running a migration yet. These pages compare docs.page to common alternatives at a thematic level so you can decide which model fits your team.

docs.page is built for **open-source maintainers, SDK authors, and product teams** who publish documentation in **public GitHub repositories**. That focus keeps docs next to code, makes agent surfaces like [llms.txt](/ai-agents/llms-txt) and [MCP](/ai-agents/mcp-server) available without extra setup, and removes build pipelines and hosting projects from the publish loop.
