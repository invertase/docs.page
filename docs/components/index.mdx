docs.page registers built-in MDX shortcode components during compilation. Use them in any `.mdx` file under `docs/`.

## Available components

| Component | Purpose |
| --- | --- |
| `Accordion` | Collapsible content sections |
| `Callout` | Info, warning, and error callouts |
| `Card` / `Cards` | Linked content cards |
| `CodeGroup` | Tabbed code examples |
| `Image` | Documentation images with optional zoom |
| `Icon` | Inline icons |
| `Property` | Configuration field reference blocks |
| `Steps` / `Step` | Sequential procedure steps |
| `Tabs` / `TabItem` | Tabbed content panels |
| `Tweet` | Embedded tweets |
| `Video` | Generic video embed |
| `YouTube` | YouTube embed |
| `Vimeo` | Vimeo embed |

## Accordion

```mdx
<Accordion title="Advanced options">
  Content hidden until expanded.
</Accordion>
```

## Callout

```mdx
<Callout type="info">
  Informational note for readers.
</Callout>

<Callout type="warning">
  Important caution before proceeding.
</Callout>
```

## Steps

```mdx
<Steps>
  <Step title="Install dependencies">
    Run `npm install`.
  </Step>
  <Step title="Start the server">
    Run `npm run dev`.
  </Step>
</Steps>
```

## CodeGroup

Wrap equivalent commands in a `CodeGroup` with labeled fenced code blocks for tabbed switching.

## Property

For reference pages, document configuration fields:

```mdx
<Property name="theme.primary" type="hex color">
  Custom primary brand color.
</Property>
```

## Tabs

Use `Tabs` and `TabItem` for tabbed panels. Put fenced code blocks inside each tab item:

```mdx
<Tabs>
  <TabItem label="npm" value="npm">
    npm install command here
  </TabItem>
  <TabItem label="pnpm" value="pnpm">
    pnpm install command here
  </TabItem>
</Tabs>
```
