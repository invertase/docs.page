docs.page builds a per-repository search index from MDX content and serves it for client-side FlexSearch. Readers open search with **⌘K** (or the search dialog UI).

## How it works

1. At bundle time, docs.page extracts searchable text from all MDX pages.
2. The index is published at `GET /{owner}/{repo}/search.json`.
3. The client loads the index into a Web Worker and queries via FlexSearch.

No additional configuration is required — search is enabled by default for all sites.

## Search index API

Integrators can fetch the raw index:

```
GET /{owner}/{repo}/search.json
```

The payload is FlexSearch-compatible JSON built by `buildDocsFlexSearchIndex`.

Ref-aware URLs apply when viewing branch, PR, or commit previews:

```
GET /{owner}/{repo}~{ref}/search.json
```

## Algolia DocSearch

The `search.docsearch` field in `docs.json` accepts Algolia credentials (`appId`, `apiKey`, `indexName`).

<Warning>
**Schema only.** DocSearch configuration is defined in the schema and migrated from v1 configs, but the search UI implements FlexSearch only. Algolia credentials in `docs.json` have no effect on the built-in search dialog today.
</Warning>
