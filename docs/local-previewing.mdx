Browser-based preview at /preview that reads a local directory via the File System Access API, persists the handle in IndexedDB, and POSTs content to the preview API for live rendering.

## Open preview mode

1. Visit [docs.page/preview](https://docs.page/preview)
2. Click **Select directory** and choose your project folder
3. Browse pages — changes reload as you edit files

Requires a browser with the [File System Access API](https://developer.mozilla.org/en-US/docs/Web/API/File_System_Access_API) (Chrome, Edge).

## Validation

The preview UI runs the same `check()` validation as the CLI. Fix errors shown in the toolbar.

See [CLI check](/cli/check).
## Implementation

- `website/src/layouts/preview/index.tsx` — preview UI
- `website/src/layouts/preview/utils.ts` — File System API and API calls
- `website/src/pages/preview/[[...path]].tsx` — route
