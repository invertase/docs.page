docs.page exposes documentation to AI agents and LLM pipelines through an MCP HTTP server and markdown index exports.

## MCP server

Each documentation site can expose a Model Context Protocol server at:

```
POST /{owner}/{repo}/mcp
GET  /{owner}/{repo}/mcp
```

Enable or disable via `docs.json`:

```json
{
  "mcp": {
    "enabled": true
  }
}
```

`mcp.enabled` defaults to `true`.

### Tools

| Tool | Description |
| --- | --- |
| `read_doc_page` | Read a `.md` or `.mdx` source file by path |
| `list_doc_files` | List all `.mdx` pages in the repository context |

### Resources

| Resource | URI | Description |
| --- | --- | --- |
| Config schema | `docs-page://schema/config` | JSON Schema for `docs.json` |
| Agent skills | `docs-page://skills/{path}` | `SKILL.md` files from `.agents/skills/*/SKILL.md` |

Skill resources are discovered by scanning the repository tree for `.agents/skills/*/SKILL.md` files.

## llms.txt index

`GET /{owner}/{repo}/llms.txt` returns a markdown index listing all documentation pages with titles, descriptions, and absolute URLs. Designed for LLM context ingestion.

Example entry format:

```markdown
- [Getting Started](https://docs.page/org/repo/getting-started): Scaffold your first site
```

## llms-full.txt export

`GET /{owner}/{repo}/llms-full.txt` returns the full aggregated content of all documentation pages in a single markdown file for bulk LLM ingestion.

## Raw markdown endpoints

Individual page source is also available:

```
GET /{owner}/{repo}/{path}.md
GET /{owner}/{repo}/{path}.mdx
```
