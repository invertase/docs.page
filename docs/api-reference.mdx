docs.page exposes HTTP endpoints for integrators, operators, and AI tooling. All per-repo paths support ref segments (`~{branch|pr|sha}`) and vanity/custom-domain equivalents.

## Document bundle

Returns parsed config, MDX IR, headings, frontmatter, and repository metadata for a single page.

```
GET /api/bundle?owner={owner}&repository={repo}&path={path}&ref={ref}
```

| Parameter | Required | Description |
| --- | --- | --- |
| `owner` | yes | GitHub owner |
| `repository` | yes | Repository name |
| `path` | yes | Page path under `docs/` |
| `ref` | no | Branch, PR number, or commit SHA |

## Config JSON Schema

Exposes the `docs.json` configuration schema for editors and tooling.

```
GET /api/schema.json
GET /schema.json
```

## Search index {#search-index}

FlexSearch-compatible search payload for a repository.

```
GET /{owner}/{repo}/search.json
```

## Raw markdown

Unrendered MDX or markdown source for a page.

```
GET /{owner}/{repo}/{path}.md
GET /{owner}/{repo}/{path}.mdx
```

## LLM exports

| Endpoint | Description |
| --- | --- |
| `GET /{owner}/{repo}/llms.txt` | Page index with titles, descriptions, and URLs |
| `GET /{owner}/{repo}/llms-full.txt` | Full aggregated page content |

## SEO

| Endpoint | Description |
| --- | --- |
| `GET /{owner}/{repo}/sitemap.xml` | Sitemap with canonical URLs |
| `GET /{owner}/{repo}/robots.txt` | Robots file referencing the sitemap |

Canonical URLs respect vanity subdomain and custom domain publishing context.

## MCP server

Model Context Protocol HTTP transport.

```
POST /{owner}/{repo}/mcp
GET  /{owner}/{repo}/mcp
```

Tools: `read_doc_page`, `list_doc_files`. Resources: config schema, agent skill files.

## Agent {#agent}

| Method | Endpoint | Description |
| --- | --- | --- |
| `POST` | `/api/agent` | Stream agent chat responses |
| `POST` | `/api/agent/create` | Provision agent (requires GitHub admin + JWE credentials) |
| `DELETE` | `/api/agent` | Delete agent (requires GitHub admin) |
| `GET` | `/.well-known/jwks.json` | Public JWKS for client-side credential encryption |

## Open Graph images

Dynamic social preview image generation.

```
GET /api/og?{params}
```

Parameters are constructed from page metadata. See [Social & SEO](/configuration#social-seo).

## Health check

```
GET /api/health
```

Returns a simple OK response for service health probes.
