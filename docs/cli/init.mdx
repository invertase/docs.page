---
title: docs init
description: Scaffold docs.json and optional starter docs/ MDX files in a project.
---

`docs init` creates a `docs.json` configuration file and optional starter pages under `docs/` using Handlebars templates.

## Usage

```bash
docs init [path] [options]
```

## Options

| Flag | Description |
| --- | --- |
| `--name <name>` | Project name for `docs.json` |
| `--docs` | Create starter `docs/` MDX files (default) |
| `--no-docs` | Create `docs.json` only |
| `--overwrite` | Overwrite existing files |

## Examples

Scaffold in the current directory:

```bash
npx @docs.page/cli init
```

Scaffold with a custom name, config only:

```bash
npx @docs.page/cli init --name "My API" --no-docs
```

Scaffold in a subdirectory:

```bash
npx @docs.page/cli init ./my-project
```

## Output

With `--docs` (default), the command creates:

- `docs.json`
- `docs/index.mdx`
- `docs/usage.mdx`

## Verify

```bash
npx @docs.page/cli check
```
