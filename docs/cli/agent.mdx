The `docs agent` command provisions or removes the in-docs AI agent for a repository.

## docs agent create

Encrypts provider credentials client-side and calls the agent provisioning API.

```bash
docs agent create [options]
```

### Options

| Flag | Description |
| --- | --- |
| `--repo <org/name>` | Target repository (defaults to current repo) |
| `--provider <name>` | LLM provider: `xai`, `openai`, `anthropic`, or `google` |
| `--apikey <key>` | Provider API key (encrypted before transmission) |
| `--gh-auth <token>` | GitHub token with admin access to the repo |
| `--force` | Overwrite an existing agent |

### Example

```bash
docs agent create \
  --repo invertase/docs.page \
  --provider openai \
  --apikey sk-...
```

The CLI fetches `GET /.well-known/jwks.json`, encrypts the API key with JWE, and calls `POST /api/agent/create`.

Set `DOCS_PAGE_API_BASE` to target a non-default API host.

### After creation

Add the returned `agent.key` to `docs.json`:

```json
{
  "agent": {
    "key": "returned-key-from-create"
  }
}
```

## docs agent delete

Removes the agent for a repository.

```bash
docs agent delete [options]
```

### Options

| Flag | Description |
| --- | --- |
| `--repo <org/name>` | Target repository |
| `--gh-auth <token>` | GitHub token with admin access |

### Example

```bash
docs agent delete --repo invertase/docs.page
```

Calls `DELETE /api/agent` after GitHub admin verification.

## Troubleshooting

| Symptom | Likely cause | Fix |
| --- | --- | --- |
| Permission denied | Token lacks admin access | Use a GitHub token with admin on the repo |
| JWKS fetch fails | Wrong API base | Set `DOCS_PAGE_API_BASE` correctly |
| Agent already exists | Previous provisioning | Use `--force` to overwrite |
