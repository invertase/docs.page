title: docs check
description: Validate docs.json and scan MDX for broken links and missing assets.
---

`docs check` validates `docs.json` (or `docs.yaml`) and scans MDX files for broken internal links, missing local assets, and unreachable external links.

## Usage

```bash
docs check [path] [options]
```

## Options

| Flag | Values | Default | Description |
| --- | --- | --- | --- |
| `--external-links` | `off`, `warn`, `error` | `warn` | Check external URL reachability |
| `--internal-links` | `off`, `warn`, `error` | `error` | Check internal `href` targets |
| `--assets` | `off`, `warn`, `error` | `error` | Check local asset file existence |

## Examples

Validate the current project with defaults:

```bash
npx @docs.page/cli check
```

Skip external link checks in CI:

```bash
npx @docs.page/cli check --external-links off
```

Treat asset issues as warnings:

```bash
npx @docs.page/cli check --assets warn
```

## What it checks

1. **Configuration** — `docs.json` or `docs.yaml` parses against the current schema (including v1 migration)
2. **Internal links** — `href` values in MDX resolve to existing pages or valid anchors
3. **Assets** — relative image and file paths exist on disk
4. **External links** — HTTP HEAD requests to external URLs (when enabled)

## Verify

A successful run exits with code 0 and reports no errors at the configured severity levels.
