title: docs.json Reference
description: Configuration fields for docs.json and docs.yaml validated at parse time.
---

docs.page reads `docs.json` or `docs.yaml` from your repository root. The configuration schema is exposed at `GET /api/schema.json` and `GET /schema.json` for editor autocomplete.

Legacy v1 `docs.json` shapes are accepted and transformed into the current schema, preserving sidebar structure and common fields.

## Top-level fields

<Property name="name" type="string">
  Project name shown in the header when `header.showName` is enabled.
</Property>

<Property name="description" type="string">
  Site description used in metadata and MCP server descriptors.
</Property>

<Property name="sidebar" type="array or locale map">
  Navigation tree. Use an array for a single locale, or a locale-keyed object for multi-locale sites. See [Navigation](/navigation).
</Property>

<Property name="tabs" type="Tab[]">
  Top-level documentation tabs with optional locale scoping. See [Navigation](/navigation#tabs).
</Property>

<Property name="variables" type="record">
  Key-value map for double-brace variable substitution in MDX content. See [Writing Content](/writing-content#variables).
</Property>

## Theme

<Property name="theme.defaultTheme" type="light or dark">
  Default color mode for first-time visitors.
</Property>

<Property name="theme.preset" type="string">
  Shadcn preset name. Resolved to CSS theme variables at parse time.
</Property>

<Property name="theme.primary" type="hex color">
  Custom primary brand color (`#RRGGBB`).
</Property>

<Property name="theme.primaryLight" type="hex color">
  Primary color override for light mode.
</Property>

<Property name="theme.primaryDark" type="hex color">
  Primary color override for dark mode.
</Property>

<Property name="theme.backgroundLight" type="hex color">
  Background color for light mode.
</Property>

<Property name="theme.backgroundDark" type="hex color">
  Background color for dark mode.
</Property>

---

## Header, logo, and favicon

<Property name="header.showName" type="boolean">
  Show the project name in the site header. Default: `true`.
</Property>

<Property name="header.showThemeToggle" type="boolean">
  Show the light/dark theme toggle. Default: `true`.
</Property>

<Property name="header.showGitHubCard" type="boolean">
  Show the GitHub repository card in the header. Default: `true`.
</Property>

<Property name="header.links" type="array">
  Custom header links. Each item has `title`, `href`, optional `cta` (boolean), and optional `locale`.
</Property>

<Property name="logo.light" type="string">
  Logo URL for light mode.
</Property>

<Property name="logo.dark" type="string">
  Logo URL for dark mode.
</Property>

<Property name="favicon" type="string">
  Favicon URL.
</Property>

---

## Content display

<Property name="content.headerDepth" type="number">
  Maximum heading depth included in the table of contents. Default: `3`.
</Property>

<Property name="content.zoomImages" type="boolean">
  Enable click-to-zoom on documentation images. Default: `false`.
</Property>

<Property name="content.automaticallyInferNextPrevious" type="boolean">
  Infer previous/next links from sidebar order when frontmatter does not override. Default: `true`.
</Property>

<Property name="content.showPageTitle" type="boolean">
  Show the page title above content. Default: `true`.
</Property>

<Property name="content.showPageImage" type="boolean">
  Show the page `image` frontmatter field when set. Default: `true`.
</Property>

---

## Search

<Property name="search.docsearch" type="object">
  Algolia DocSearch credentials (`appId`, `apiKey`, `indexName`).
</Property>

<Warning>
**Schema only.** DocSearch fields are defined in the configuration schema but no client-side Algolia integration is implemented. Built-in search uses FlexSearch. See [Search](/search).
</Warning>

---

## Agent

<Property name="agent.key" type="string">
  Agent provisioning key. Required for the in-docs AI chat panel.
</Property>

<Property name="agent.placeholder" type="string">
  Placeholder text in the agent input field.
</Property>

<Property name="agent.questions" type="string[]">
  Suggested starter questions shown in the agent panel.
</Property>

<Property name="agent.limits.ip" type="number">
  Maximum agent API requests per client IP per window. Default: `200`.
</Property>

<Property name="agent.limits.repo" type="number">
  Maximum agent API requests per repository per window. Default: `10000`.
</Property>

See [Documentation Agent](/agent) for setup.

---

## MCP

<Property name="mcp.enabled" type="boolean">
  Enable the per-repository MCP HTTP server at `/{owner}/{repo}/mcp`. Default: `true`.
</Property>

See [MCP & LLM Exports](/mcp-and-llms).

---

## Social & SEO

<Property name="social" type="object">
  Social profile links (e.g. `github`, `x`, `discord`).
</Property>

<Property name="og" type="object">
  Open Graph image configuration. Dynamic images are generated at `GET /api/og`.
</Property>

<Property name="seo.noindex" type="boolean">
  Set `noindex` on documentation pages.
</Property>

<Property name="socialPreview" type="string | false">
  Custom social preview image URL, or `false` to disable.
</Property>

---

## Analytics scripts

<Property name="scripts.googleTagManager" type="string">
  Google Tag Manager container ID.
</Property>

<Property name="scripts.googleAnalytics" type="string">
  Google Analytics measurement ID.
</Property>

<Property name="scripts.plausible" type="string | boolean">
  Plausible domain string or `true` for default domain-based tracking.
</Property>

## Anchors

<Property name="anchors" type="Anchor[]">
  Icon links rendered in the documentation header with optional `locale` and `tab` scoping.
</Property>
