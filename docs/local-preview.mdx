---
title: Local Preview
description: Edit MDX locally with hot reload via the docs.page preview page.
---

Use local preview to iterate on documentation without pushing to GitHub. The preview page connects to a WebSocket dev server in your project for live MDX editing.

## Before you begin

- A `docs.json` file and `docs/` directory in your project
- A compatible local preview WebSocket server running in the project (started by your dev tooling)

## Steps

1. **Start your local preview server** in the project directory. Your editor or dev tooling should expose the WebSocket endpoint the preview client expects.

2. **Open the preview page** at [https://docs.page/preview](https://docs.page/preview).

3. **Select your project directory** when prompted. The preview client loads `docs.json` and MDX files from disk.

4. **Edit MDX files** in your editor. Changes hot-reload in the preview without a GitHub fetch.

<Callout type="warning">
Local preview does not serve content from GitHub. Published URLs at `docs.page/{owner}/{repo}` always read from the remote repository.
</Callout>

## Troubleshooting

| Symptom | Likely cause | Fix |
| --- | --- | --- |
| Preview does not connect | WebSocket server not running | Start the local preview server in your project |
| Stale content | Preview cache | Refresh the preview page |
| Missing pages | `docs.json` sidebar mismatch | Ensure each `href` maps to a file under `docs/` |
