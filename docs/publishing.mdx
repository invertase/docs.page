title: Publishing

docs.page publishes documentation from **public** GitHub repositories. Configure your URL strategy with vanity subdomains, custom domains, or ref-based previews.

## Public repository requirement

docs.page blocks private repositories with a **403** error during bundling. Your documentation source must be publicly accessible on GitHub.

## Canonical URLs

The default published URL is:

```
https://docs.page/{owner}/{repo}/{path}
```

- `{owner}` — GitHub user or organization
- `{repo}` — repository name
- `{path}` — page path under `docs/` (omit for the index page)

## Branch, PR, and commit previews

Append `~{ref}` after the repository name to pin documentation to a specific ref:

| Ref type | URL pattern | Example |
| --- | --- | --- |
| Branch | `/{owner}/{repo}~{branch}/{path}` | `/invertase/docs.page~main/configuration` |
| Pull request | `/{owner}/{repo}~{pr-number}/{path}` | `/invertase/docs.page~42/getting-started` |
| Commit SHA | `/{owner}/{repo}~{sha}/{path}` | `/invertase/docs.page~abc123f/index` |

A ref badge appears in the UI when viewing non-default refs.

Pinned commit refs receive longer CDN cache TTLs for bundle JSON than mutable branches.

## Vanity subdomains

Serve docs at `{owner}.docs.page` by rewriting requests to the canonical `docs.page/{owner}/{repo}` path.

```
https://invertase.docs.page/docs.page/getting-started
```

Reserved subdomains (`docs`, `www`, `staging`) cannot be used as vanity owners.

## Custom domains

Map a custom hostname to an owner/repo pair via an external lookup service. docs.page serves the site in custom-domain request mode and sets canonical URLs accordingly.

Custom domain setup is managed outside the repository. Contact docs.page support or your account dashboard to configure DNS.

## Raw markdown export

Request unrendered source by appending `.md` or `.mdx` to any page path:

```
https://docs.page/{owner}/{repo}/getting-started.md
```

Content negotiation supports markdown clients.

## SEO exports

Each site exposes machine-readable indexes:

| Endpoint | Purpose |
| --- | --- |
| `/{owner}/{repo}/sitemap.xml` | Sitemap with canonical URLs |
| `/{owner}/{repo}/robots.txt` | Robots file referencing the sitemap |

Sitemap URLs respect vanity and custom domain publishing context.
